# Data Safety — Nitido

Last updated: 2026-05-03

## Personal Data Collected

| Data Type | Purpose | Collection | Optional | Encrypted in transit | Encrypted at rest |
|---|---|---|---|---|---|
| Financial transactions | App functionality (manual entry + auto-import) | Local-first, sync to user's Firestore | Yes (sync can be disabled) | Yes (HTTPS) | Yes (Firestore-managed) |
| User account info (email, name) | Authentication | Firebase Auth | No (required for sync) | Yes | Yes |
| App usage analytics | App improvement | Firebase Analytics | Yes (can opt-out via Settings) | Yes | Yes |
| Crash reports | App stability | Firebase Crashlytics | Yes (disabled in debug, can opt-out) | Yes | Yes |
| Device ID | Crashlytics + Analytics correlation | Auto-generated by Firebase | No | Yes | Yes |
| SMS messages | Auto-import from banking SMS | User-granted READ_SMS / RECEIVE_SMS | Yes (permission request) | N/A (processed locally) | N/A (not stored) |
| Notifications | Auto-import from banking notifications | User-granted NotificationListenerService | Yes (user must enable in system settings) | N/A (processed locally) | N/A (not stored raw) |
| Audio recordings | AI dictation (voice input) | User-granted RECORD_AUDIO | Yes (button-initiated) | N/A (on-device speech recognition) | N/A (not stored) |

## Data NOT Collected

- Location (no GPS, no IP geolocation)
- Contacts
- Calendar
- Photos/media (image_picker only opens system picker, app does not store)
- Health data
- Browsing history
- Social profiles
- Files stored on device (except app-internal database)

## Data Processing for AI Features

When AI-powered auto-import or chat features are used, transaction text may be sent to:

- **Nexus AI Gateway** (`api.ramsesdb.tech`, default provider) — managed by the app developer. No logging beyond rate-limit metadata. Text is processed ephemerally.
- **User's own provider key (BYOK)**: OpenAI, Anthropic, or Gemini — direct connection. The app developer has no visibility into these requests.

AI text processing is always initiated by the user (auto-import is opt-in, chat is manual). Processed text is NOT stored on LLM provider servers (Nexus: ephemeral; BYOK: depends on provider policy — users should review their provider's privacy policy).

## Data Sales

- We do **NOT** sell user data to third parties.
- We do **NOT** share user data with advertisers.

## Data Security

- Data in transit: HTTPS (TLS)
- Data at rest: Firestore-managed encryption; local DB stored in app-private storage
- API keys (BYOK): encrypted locally with `FlutterSecureStorage`
- Android backup: disabled (`android:allowBackup="false"`) — financial data is NOT backed up to Google Drive

## User Rights

- **Export**: Settings → Export → CSV / JSON
- **Delete account**: Settings → Account → Delete account (deletes all Firestore data)
- **Opt-out from analytics**: Settings → Privacy → Disable analytics
- **Data portability**: Export any time, in standard formats

## Play Store Data Safety Form Reference

When filling the Data Safety form in Google Play Console:

| Question | Answer |
|---|---|
| Financial info | Collected, encrypted in transit |
| Personal info (email, name) | Collected, encrypted in transit |
| App activity (analytics) | Collected, optional |
| Device or other IDs | Collected |
| SMS | Collected, ephemeral (processed locally, not stored) |
| Notifications | Collected, ephemeral (processed locally, not stored) |
| Audio | Collected, optional, ephemeral (on-device) |
| Data shared with third parties | Yes (Firebase Analytics, Crashlytics, Auth, Firestore; LLM providers when using BYOK) |
| Data can be deleted | Yes (account deletion + export) |
| Encryption in transit | Yes |
